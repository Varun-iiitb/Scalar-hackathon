# IsoSync — Isochronous Translation RL Environment

## Problem

Current AI dubbing systems translate correctly but ignore timing. The result is dubbed audio that doesn't fit the speaker's mouth movements — a critical failure for social video platforms where lip-sync authenticity drives viewer trust. IsoSync trains an LLM to translate under temporal pressure, making every word choice a timing decision.

## Why This Matters for Reels

Meta's Reels dubbing pipeline serves billions of short-form videos across Hindi, Portuguese, and Spanish markets. Each dubbed segment must fit within the original speaker's time window — even a 0.3-second overrun breaks lip sync and destroys immersion. IsoSync directly models this constraint as an RL problem, training agents that learn the tradeoff between semantic fidelity and spoken duration — exactly the problem Meta's dubbing team faces at scale.

---

## Environment Design

### The Cumulative Budget Mechanic

Every episode has a **shared timing budget** = sum of all segment durations + a small slack per segment. When the agent's translation runs longer than the original, the overrun is deducted from the shared pool — future segments get a tighter budget. This creates genuine sequential dependency: a sloppy translation early in the episode makes every subsequent segment harder to fit.

```
Episode budget = Σ(segment durations) + slack × N segments
Budget deficit += max(0, estimated_duration − target_duration)  per step
Episode terminates early if deficit > 3.0 seconds
```

### What the Agent Sees (Observation)

```
You are a professional video dubbing translator.

Translate the following English segment into Hindi.

Original: Add a pinch of salt to the boiling water.
Time window: 2.5 seconds
Max syllables: 11
Budget remaining: 12.3s for 4 segments
Budget status: ON TRACK
Locale: Mumbai

Rules:
- Your translation MUST fit within 2.5 seconds when spoken
- Preserve the core meaning
- Sound natural in Mumbai
- Do not copy the English text

Respond with ONLY the translation. No explanation.
```

### What the Agent Does (Action)

```
{"translation": "उबलते पानी में एक चुटकी नमक डालें।"}
```

### How the Episode Ends

- **Completed** — all N segments translated successfully
- **Early termination** — cumulative budget deficit exceeds 3.0 seconds

---

## Reward Functions

| Component | Weight | How it's computed |
|---|---|---|
| **Timing** | 35% | `max(0, 1 − |estimated − target| / target)` — syllable count via pyphen / Unicode |
| **Semantic** | 35% | chrF score (sacrebleu) vs reference translation, normalised to [0, 1] |
| **Budget** | 20% | `1.0` if deficit < 0.2s, `0.0` if > 2.0s, linear interpolation between |
| **Locale** | 10% | Fraction of words in locale-specific approved wordlist (Mumbai Hindi / Brazil PT) |
| **Anti-hacking** | additive | −1.0 copy-paste, −1.0 too short, −0.5 absurd overrun, −0.3 wrong script |

**Reward range: −1.0 to 1.0**

## Curriculum

| Level | Segments | Duration Slack | Locale Constraints | Episodes |
|---|---|---|---|---|
| 1 — Easy | 5 | 0.5s per segment | Off | 0 – 200 |
| 2 — Medium | 8 | 0.2s per segment | Off | 200 – 500 |
| 3 — Hard | 10 | 0.1s per segment | **On** | 500 – 1000 |

---

## Dataset

50 hardcoded English sentences across cooking, travel, and lifestyle topics (typical Reels content), each with:
- Reference Hindi translation (Mumbai locale)
- Reference Portuguese translation (Brazil locale)
- Calibrated original duration (1.5 – 4.0 seconds)

Episodes are generated by sampling N sentences from the bank with a small random duration jitter for variety.

## API Reference

The environment is served as a REST API (FastAPI, port 7860).

### `POST /reset`
```json
{"level": 1, "language": "Hindi"}
```
Returns `{"observation": "...", "state": {...}}`

### `POST /step`
```json
{"translation": "उबलते पानी में एक चुटकी नमक डालें।"}
```
Returns `{"observation": "...", "reward": 0.82, "done": false, "info": {...}}`

### `GET /state` — full environment state
### `GET /info` — metadata
### `GET /health` — liveness probe
### `GET /docs` — interactive Swagger UI

## Stack

| Component | Library |
|---|---|
| LLM backbone | Qwen2.5-1.5B-Instruct (4-bit via Unsloth) |
| RL algorithm | GRPO (TRL `GRPOTrainer`) |
| PEFT | LoRA rank-8, alpha-16 |
| Syllable counting | pyphen (PT) + Devanagari Unicode heuristic (HI) |
| Semantic scoring | chrF via sacrebleu |
| Server | FastAPI + Uvicorn |
| Deployment | HuggingFace Spaces (Docker) |
